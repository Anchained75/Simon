# Cursor IDE & GitHub Setup Documentation

## Overview
This project documents the setup and configuration of a development environment using Cursor IDE, AI-powered coding extensions, and GitHub for version control. The objective was to successfully install required tools, integrate them, and publish a structured repository documenting the process.

---

## Tools & Technologies

- Cursor IDE – An AI-powered code editor designed to streamline development workflows and integrate advanced coding assistants.
- Claude Code Extension – Provides AI-assisted coding, reasoning, and debugging support within the IDE.
- Codex Extension – Enables additional AI-powered code generation and assistance capabilities.
- GitHub – A version control platform used to host and manage code repositories publicly.

---

## Setup Process

1. Downloaded and installed Cursor IDE from the official website.
2. Opened Cursor and navigated to the Extensions marketplace.
3. Searched for and installed the Claude Code extension, then authenticated the account.
4. Searched for and installed the Codex extension, followed by login and activation.
5. Created a new public repository on GitHub.
6. Opened the repository within Cursor IDE.
7. Created a `README.md` file to document the setup process.
8. Staged changes, committed the file, and pushed the repository to GitHub.

---

## Challenges & Problem-Solving

### Issue 1: Extension Installation Confusion
- **Problem:** Initially unclear whether extensions were fully installed or required additional activation.
- **Cause:** Lack of familiarity with Cursor’s extension workflow.
- **Solution:** Verified installation status within the Extensions panel and completed login/authentication steps to fully enable functionality.

### Issue 2: GitHub Repository Integration
- **Problem:** Difficulty confirming whether the local project was properly linked to the remote GitHub repository.
- **Cause:** First-time interaction with repository linking inside an IDE.
- **Solution:** Followed standard Git workflow: initialized repository, staged changes, committed, and pushed to confirm successful synchronization.

### Issue 3: Markdown File Formatting
- **Problem:** Needed to ensure the README was clean, structured, and readable.
- **Cause:** Limited prior use of Markdown formatting.
- **Solution:** Referenced Markdown documentation and applied headings, lists, and spacing to improve clarity and presentation.
